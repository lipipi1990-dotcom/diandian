🏗️ 三层系统架构
🧩

策略层 — Strategy Layer

每个策略是独立的 Python 文件，继承 BaseStrategy 接口。输出 Signal 对象包含具体价位（入场价、止损、止盈），而非百分比。新增策略只需创建文件并在 config 中添加一行配置。

↕
⚙️

引擎层 — Engine Layer

核心包含三大模块：DataHub（WebSocket 数据接收与智能分发，相同交易对只订阅一次）、VirtualBroker（虚拟撮合引擎，模拟手续费、滑点、杠杆和强平）、StrategyRunner（策略实例调度器，独立账户互不影响）。

↕
📊

展示层 — Dashboard Layer

FastAPI + Jinja2 模板驱动的管理页面。首页为策略排行榜，支持点击进入详情查看资金曲线、交易记录、持仓状态和开仓原因。通过 ngrok 实现远程访问。

🔄 策略添加流程

发现策略

从 GitHub、社交媒体、交易社区发现策略想法——可以是 Pine Script、Python 代码或纯文字描述

AI 翻译

将策略描述发给 Claude，说"帮我翻译成 strategy-lab 的标准格式"，自动生成符合 BaseStrategy 接口的 .py 文件

即插即用

将生成的文件放入 strategies/ 目录，在 config.py 中添加一条实例配置，重启系统即开始模拟运行

数据验证

策略至少跑满 2 周、完成 30+ 笔交易，积累有统计意义的数据样本

毕业筛选

总收益为正、最大回撤可接受（<15%）的策略标记为"达标"，进入候选池

上实盘

手动将策略文件复制到独立的实盘系统，开始真实交易。实验室继续运行模拟作为对照

策略实例配置 — config.py

Python STRATEGY_INSTANCES = [ { "strategy": "ema_stochrsi", # 策略文件名 "timeframe": "15m", # 时间框架 "pair": "BTC/USDT", # 交易对 "leverage": 20, # 杠杆倍数 "initial_balance": 1000, # 虚拟初始资金 (U) "params": { "ema_period": 20, "atr_multiplier": 1.5 } }, { "strategy": "bollinger_rsi", "timeframe": "1h", "pair": "ETH/USDT", "leverage": 10, "initial_balance": 1000, "params": {"bb_period": 20, "rsi_period": 14} } ]

全局配置

Python EXCHANGE = "hyperliquid" # 数据源交易所 TAKER_FEE = 0.0005 # 0.05% 吃单手续费 MAKER_FEE = 0.0002 # 0.02% 挂单手续费 SLIPPAGE = 0.0001 # 0.01% 滑点模拟 SNAPSHOT = "daily" # 每日快照用于盈亏曲线

Signal 对象字段

dataclass @dataclass class Signal: direction: str # "long" 或 "short" entry_price: float # 70636.60（具体入场价） stop_loss: float # 70995.83（基于 ATR 计算） take_profit_1: float # 70097.75（TP1 具体价位） take_profit_2: float | None # 可选的 TP2 confidence: float # 0.0 ~ 1.0 策略自评信心度 reason: str # 开仓原因，用于日志和页面展示
